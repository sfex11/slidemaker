# slide-maker 개선안

## 목표
- 사용자 입력이 달라도 일관된 품질의 슬라이드를 안정적으로 생성
- 실패를 줄이고, 실패 시 빠르게 복구(Self-healing)
- 결과를 측정 가능한 품질 지표로 관리

## 핵심 개선 축
1. 스키마 우선 생성
- 허용 타입(title, card-grid, comparison, timeline, quote, table) 외 출력 차단
- content 구조 정규화로 렌더 실패 가능성 최소화

2. 품질 평가 루프
- 구조 점수: 타이틀/슬라이드 수/핵심 본문 구조
- 가독성 점수: 문장 길이, 항목 과밀, 빈 콘텐츠 방지
- 다양성 점수: 타입 다양성(단일 타입 편중 완화)

3. 자동 보정(Self-healing)
- 슬라이드 수 부족 시 요약/타임라인/인용 슬라이드 자동 보강
- 품질 점수가 임계치 미만이면 폴백 생성으로 재시도
- 보정 후 재평가하여 최종 품질 확보

4. 산출물 가시성
- 생성 결과에 품질 리포트(점수/이슈) 포함
- 프로젝트 설명에 품질 스냅샷 기록

## 이번 반영 내용
- 서버에 `품질 평가 + 자동 보정 + 임계치 기반 폴백` 로직 추가
- API 응답에 품질 리포트 포함
- 프로젝트 설명(description)에 품질 점수 저장

## 다음 단계 (권장)
1. UI에서 품질 리포트 노출(점수 + 개선 포인트)
2. 생성 전 아웃라인 승인 단계 추가(Human-in-the-loop 강화)
3. 템플릿 계약 테스트를 CI에 추가
4. 품질 점수 추세를 프로젝트별로 시계열 저장/분석
